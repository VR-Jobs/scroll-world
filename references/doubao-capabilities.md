# 豆包模型能力边界（依据用户提供的官方文档，核对于 2026-07-14）

这里记录影响流水线设计的稳定能力，不记录会变化的价格。使用前仍以火山方舟控制台的当前
模型列表与计费页为准。

## Seedream 5.0 Pro

- 模型 ID：`doubao-seedream-5-0-pro-260628`。
- 文生图、单/多参考图；最多 10 张参考图。
- 1K/2K，支持 PNG/JPEG；本技能默认 2K、URL 返回、关闭水印。
- 参考图可用 JPEG/PNG/WebP/BMP/TIFF/GIF/HEIC/HEIF；单图 ≤30MB、≤36MP。
- 自定义尺寸总像素和长宽比有约束；滚动站优先用官方 2K 16:9/9:16 档。
- 输出 URL 只有临时有效期（文档提示 24 小时级），成功后必须立即下载本地。
- Pro 不支持 stream；不要向请求添加未经模型支持的流式参数。

## Seedance 2.0 系列

- Full：`doubao-seedance-2-0-260128`，480p/720p/1080p/4K。
- Fast：`doubao-seedance-2-0-fast-260128`，480p/720p。
- Mini：`doubao-seedance-2-0-mini-260615`，480p/720p。
- 支持文生、首帧、首尾帧、多模态参考、编辑/延长、生成音频和返回尾帧。
- 比例：21:9、16:9、4:3、1:1、3:4、9:16；时长 4–15 秒；输出 MP4。
- 多模态上限：图片 0–9、视频 0–3、音频 0–3。不能只传文本+音频或纯音频。
- 图片角色包括 `first_frame`、`last_frame`、`reference_image`；严格端点约束要用 role，
  不要仅在 prompt 里说“这是首帧/尾帧”。
- prompt 通过“图片1/视频1/音频1”按输入顺序引用；不要在 prompt 使用 Asset ID。
- 查询成功结果的 `usage.completion_tokens`（部分 SDK 映射为 `Usage.CompletionTokens`）是
  视频计费用量依据；轮询本身不是新生成。
- Full 4K 是 HEVC 10-bit，浏览器兼容性受限；网页交付转 H.264/yuv420p。
- 首尾帧尺寸/比例不一致会导致跳变或拒绝；同一条链不混用 16:9 与 9:16。
- 成功视频/尾帧 URL 是临时交付，任务完成后立即下载。

## 文档没有提供的内容

用户提供的两份模型使用文档没有单价表。Seedream 响应可能返回 output tokens，但仅凭 usage
字段不能断言官方按 token 而非按张计费。每个项目开始时从当前官方计费页/控制台保存带日期
的价格快照和 `billing_unit`，最终用该单位对应的真实输出数量/token 计算。
